\part{Unit Test Plan}
\section{Description}
	In this section we will present an extensive suite of test cases aimed to cover in the most efficient manner possible the Fantasy Basketball system we are to deliver. Following the paradigm of Test-Driven Development, we shall also implement the unit test cases. This will help keep the development of the system on track with the requirements imposed to the team. The test cases will be split into the four main categories of the system:
    
	\begin{itemize}
	    \item \textbf{Global:} Implementation independent tests, common to all platforms
	    \item \textbf{Desktop:} Administrative specific tests
	    \item \textbf{Web:} Professor specific tests
	    \item \textbf{Mobile:} TA/Grader specific tests
	\end{itemize}
\section{Global Plan}
    All the classes to test will have the following pattern:\\
    
    \textbf{CLASS TO TEST/NOT TO TEST}
    \begin{itemize}
        \item Reason to test/not to test
        \item attributes and related tests
    \end{itemize}
